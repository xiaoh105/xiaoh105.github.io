A-Three Indices

题意

给定一个1−n1-n1−n的排列ppp,求三元组(i,j,k)(i,j,k)(i,j,k)满足pj>pk,pj>pip_j>p_k,p_j>p_ipj​>pk​,pj​>pi​或判断其不存在。

数据范围：

T≤200,3≤n≤1000,1≤pi≤nT\leq 200,3\leq n\leq 1000,1\leq p_i\leq n T≤200,3≤n≤1000,1≤pi​≤n

题解

本质上就是求一个单调上升后单调下降的序列是否存在，直接开一个boolboolbool判断之后输出即可。复杂度O(n)O(n)O(n)

B-Universal Solution

题意

给定一个由R,S,PR,S,PR,S,P组成的字符串SSS，每个字符分别表示石头剪刀布中的石头，剪刀，布。对手将从某个位置开始环形地出手势，一共出∣S∣|S|∣S∣次。对手对于每个位置的选择是等概率的。现在你需要固定你出招的序列ccc，使这个序列的胜利次数的期望着最大。求ccc。

数据范围：

t≤1000,∣c∣=∣S∣≤∑∣S∣≤2×105t\leq 1000,|c|=|S|\leq \sum |S|\leq 2\times 10^5 t≤1000,∣c∣=∣S∣≤∑∣S∣≤2×105

题解

显然，我们可以将期望值转化为如下式子：

exp(c)=1∣S∣∑i=0∣S∣∑j=0∣S∣judge(si,cj)exp(c)=\frac{1}{|S|}\sum_{i=0}^{|S|}\sum_{j=0}^{|S|} judge(s_i,c_j) exp(c)=∣S∣1​i=0∑∣S∣​j=0∑∣S∣​judge(si​,cj​)

我们可以理解为每一个cjc_jcj​都与整个∣S∣|S|∣S∣比赛一轮，然后最大化这个赢的次数。显然，选择∣S∣|S|∣S∣中出现最多的一个手势，然后每一个cjc_jcj​都选择战胜它的那个手势，可以最大化∑\sum∑里面的值。直接输出即可。复杂度O(n)O(n)O(n)

C-Create The Teams

题意

有nnn个人，每个人都有一个技能职值aia_iai​。现在需要选将其分为一些队伍，每支队伍的人数×\times×队伍中aia_iai​的最小值必须至少为xxx。求至多能分为几队。

数据范围

t≤1000,n≤∑n≤105,1≤ai,x≤109t\leq 1000,n\leq \sum n\leq 10^5,1\leq a_i,x\leq 10^9 t≤1000,n≤∑n≤105,1≤ai​,x≤109

题解

显然，通过微扰法可证明每次选择的队伍都为连续的一段时最优。将整个序列排序，然后不停地往一个队列里加人直到满足条件后关闭这个队伍，新开一个队伍继续塞即可。复杂度O(nlog⁡n)O(n\log n)O(nlogn)。

D-Berserk And Fireball

题意

给定一个长度为nnn的无重复序列aaa和一个长度为bbb的序列bbb，现有两种法术：

  1. Fireball:可以消灭连续kkk个数字，代价为xxx；
  2. Berserk:可以使大的数字消灭相邻的小的数字，代价为yyy。

求使序列aaa变为序列bbb的最小代价，或判断其不可行。

数据范围：

1≤m≤n≤2×105,1≤x,y≤109,1≤k≤n,1≤ai,bi≤n1\leq m\leq n\leq 2\times 10^5,1\leq x,y\leq 10^9,1\leq k\leq n,1\leq a_i,b_i\leq n 1≤m≤n≤2×105,1≤x,y≤109,1≤k≤n,1≤ai​,bi​≤n

题解

由于序列不可重，所以对应模式固定，若无对应则无解。

对应后，匹配的数将不匹配的数分割成了一些段，显然不同段直接不能跨段操作。所以我们将其分开考虑。记这个段的长度为lenlenlen，最大的数为ppp。当p>lp>lp>l且p>rp>rp>r时，则最大的一个数必须被fireball干掉，所以当len<klen<klen<k时无解，否则使用一定次数的berserk（至少len mod klen\bmod klenmodk次），然后用fireball干掉其他的即可。注意必须使用一次fireball。可以通过费用O(1)O(1)O(1)计算。

当p<max(l,r)p<max(l,r)p<max(l,r)时，则两端可以一直通过berserk杀光中间的数字，也可以先用berserk再用fireball处理，同上根据费用取较小者即可。复杂度O(n)O(n)O(n)。

E-Merging Towers

题意

有mmm座塔,编号为1−n1-n1−n的圆环，每座塔上从上到下按照从小到大的顺序套了一些圆环。第iii个圆环在第tit_iti​座塔上。每次可以将一座塔上面连续的一段圆环移到另一座塔上，但是不能破坏塔的性质。记一个局势的困难度为将所有圆盘放到同一根柱子上所需的操作次数。现有mmm个询问，每次会将ai,bia_i,b_iai​,bi​所在的塔合并。求合并后的困难度。注：每次询问将会保留。

数据范围：

2≤m≤n≤2×105,ai,bi保证操作前不在同一座塔上2\leq m\leq n\leq 2\times 10^5,a_i,b_i保证操作前不在同一座塔上 2≤m≤n≤2×105,ai​,bi​保证操作前不在同一座塔上

题解

经过观察，我们发现困难度即为数对(i,i+1)(i,i+1)(i,i+1)不在同一座塔上的数量。证明显然。

接下来考虑如何维护。现有两种做法，可以用启发式合并粗暴的在线求解，也可以用KruskalKruskalKruskal重构树来进行处理。

启发式合并难度较小在此不再赘述。我们着重说明一下重构树的解法。

首先，离线，以塔为叶子节点，操作的时间戳为键值建树，则对于每一个数对(i,i+1)(i,i+1)(i,i+1)，在Ti,Ti+1T_i,T_{i+1}Ti​,Ti+1​合并前都将对答案有111的贡献。而通过查询两点的lcalcalca可以快速得到合并的时刻。于是直接跑倍增的lcalcalca即可。复杂度O(nlog⁡m)O(n\log m)O(nlogm)

啥？你说要树状数组维护贡献？其实一个差分就可以啦！

const int MAXN=200010,LOGN=20;
int n,m;
int t[MAXN];
int tot=1;
int edge[MAXN*2],nxt[MAXN*2],hd[MAXN*2];
inline void add_edge(int u,int v)
{
	edge[tot]=v,nxt[tot]=hd[u],hd[u]=tot++;
}
int LCA[MAXN*2][LOGN],d[MAXN*2];
queue<int> q;
int logn;
inline void prework()
{
	logn=log2(2*m-1)+1;
	q.push(2*m-1),d[2*m-1]=1;
	while(!q.empty())
	{
		int x=q.front();q.pop();
		for(int i=hd[x];i;i=nxt[i])
		{
			int y=edge[i];
			d[y]=d[x]+1;
			LCA[y][0]=x;
			q.push(y);
			for(int j=1;j<=logn;++j) LCA[y][j]=LCA[LCA[y][j-1]][j-1];
		}
	}
}
inline int query(int x,int y)
{
	if(d[x]<d[y]) swap(x,y);
	for(int i=logn;i>=0;--i)
	if(d[LCA[x][i]]>=d[y]) x=LCA[x][i];
	if(x==y) return x;
	for(int i=logn;i>=0;--i)
	if(LCA[x][i]!=LCA[y][i]) x=LCA[x][i],y=LCA[y][i];
	return LCA[x][0];
}

	int a,b;

	int f[MAXN*2];
	inline void init()
	{
		for(int i=1;i<=2*n;++i) f[i]=i;
	}
	int find(int x)
	{
		return f[x]=(x==f[x])? f[x]:find(f[x]);
	}
}g;
int ans[MAXN];
int main()
{

	{

		add_edge(m+i,g.find(a[i].a)),add_edge(m+i,g.find(a[i].b));
		g.f[g.find(a[i].a)]=g.f[g.find(a[i].b)]=m+i;
	}
	prework();
	for(int i=1;i<n;++i)
	{
		int x=t[i],y=t[i+1];
		int tmp=query(x,y);
		++ans[0],--ans[max(tmp-m,0)];
	}
	for(int i=1;i<m;++i) ans[i]+=ans[i-1];
	for(int i=0;i<m;++i) write(ans[i]),putchar('\n');
	return 0;
}

F-Strange Addition

题意

现在我们重定义加法的计算方式，列竖式，对应位数相加，并将答案写在正下方，最终所有数字连在一起为答案。具体栗子如下：

给定算式的结果ccc，且有nnn位。现有mmm次操作，每次修改数位上的一个数字，求每次修改后满足a+b=ca+b=ca+b=c的ccc的数对的数量。要求对998244353998244353998244353取模

数据范围

1≤n,m≤5×1051\leq n,m\leq 5\times 10^5 1≤n,m≤5×105

题解

在无修改的情况下，我们可以考虑用线性dpdpdp来进行解决。用fif_ifi​表示构成ccc前iii位的加法算式的数量。那么每次可以在两个算式后面各加一位，使其成为ci+1c_{i+1}ci+1​或ci+1ci+2c_{i+1}c_{i+2}ci+1​ci+2​。粗暴的统计即可。

那么考虑如何使其支持修改。不妨用线段树进行维护。我们维护每个区间计算/不计算第一位，计算/不计算最后一位，共四种情况下的可能种数。每次合并时从两个子区间合并信息，若两个区间的衔接处可以构成一个ci+1ci+2c_{i+1}c_{i+2}ci+1​ci+2​，那么我们需要加上这种情况下的情况总数。具体的转移方程可以参考代码内的过程。总的复杂度为O(mlog⁡n)O(m\log n)O(mlogn)，当然常数会比较大，不过时间很充裕能够通过。

		f[p].val[0]=1,f[p].val[3]=a[l]+1;
		return;
	}
	int mid=(l+r)>>1;
	build(lp,l,mid),build(rp,mid+1,r);
	pushup(p);
}
void modify(int p,int x,int y)
{
	if(f[p].l==f[p].r)
	{
		f[p].val[3]=y+1;
		return;
	}
	int mid=(f[p].l+f[p].r)>>1;
	if(x<=mid) modify(lp,x,y);
	else modify(rp,x,y);
	pushup(p);
}
int main()
{
	read(n),read(m);
	scanf("%s",s+1);
	for(int i=1;i<=n;++i) a[i]=s[i]-'0';
	build(1,1,n);
	for(int i=1;i<=m;++i)
	{
		int x,y;
		read(x),read(y);
		a[x]=y;
		modify(1,x,y);
		write(f[1].val[3]),putchar('\n');
	}
	return 0;
}

G-Circular Dungeon

题意

给定nnn个正整数cic_ici​，表示nnn个金库的价值。有两种金库：

  1. regular：玩家通过会拿取该点的价值后进入下一个点；

  2. mimic：玩家会立刻死亡。

现在你需要将所有金库围成一个圈，第iii个金库仅能与通向第i+1i+1i+1个，你可以任意的安排金库的位置和类型。玩家会从任意一个点进入，一直沿着唯一的路径走知道死亡。现要求你对于k∈[1,n]k\in [1,n]k∈[1,n]，求恰有kkk个mimic类型的金库时，玩家获得价值期望的最小值。

数据范围

2≤n≤3×105,1≤ci≤1062\leq n\leq 3\times 10^5,1\leq c_i\leq 10^6 2≤n≤3×105,1≤ci​≤106

题解

所有的期望问题都可以转化为求所有情况总和的问题，我们现在不妨进行一次转化，将其变为统计总和。

接下来我们将所有金库按照被mimic分隔的方式分为一些段，那么对于每个长度为lenlenlen的段，其中金库的贡献系数分别为1,2,3,…,len1,2,3,\dots,len1,2,3,…,len。接下来我们不妨以此为基础，证明几个引理。

通过微扰法可以证明：对于任意两个段，若它们的长度差不小于222，则通过将一个数从大的移至小的可以使答案更小，因为所有段的长度都应至多相差1。

通过最简单的贪心可以证明，对于一个段内部，一定将所有价值的金库从大到小排序，然后按照1,2,3,…,len1,2,3,\dots,len1,2,3,…,len的顺序给其加上系数会使答案最小。

接下来我们便可以根据这两个引理写出一个O(n2)O(n^2)O(n2)的程序了：将最大的kkk个系数赋值为000，次大的kkk个系数为111，…\dots…，以此类推。值得注意的是，最后的几个数可能未必凑满kkk个，但是不需要特殊处理。

那么如何进行优化呢？我们注意到每次给连续的kkk个赋上相同的系数的步骤可以简化，直接利用前缀和乘上系数即可快速的计算答案。总的复杂度为∑nk=nlog⁡n\sum \frac{n}{k}=n\log n∑kn​=nlogn。可以优雅的通过本题。
