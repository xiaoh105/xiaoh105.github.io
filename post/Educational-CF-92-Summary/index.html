A-LCM Problem

题意

给定两个正整数l,rl,rl,r，求满足l≤x<y≤rl\leq x<y\leq rl≤x<y≤r且l≤lcm(x,y)≤rl\leq lcm(x,y)\leq rl≤lcm(x,y)≤r的任意一对(x,y)(x,y)(x,y)，或者判断其不存在。

数据范围

1≤t≤104,1≤l<r≤1091\leq t\leq 10^4,1\leq l<r\leq 10^9 1≤t≤104,1≤l<r≤109

题解

因为x≠yx\ne yx​=y，所以lcm(x,y)≥2xlcm(x,y)\geq 2xlcm(x,y)≥2x。所以r≥2lr\ge 2lr≥2l时有解，否则无解。构造的话直接(l,2l)(l,2l)(l,2l)即可。每个数据常数复杂度。

B-Array Walk

题意

给定nnn个正整数aia_iai​，从a1a_1a1​开始，我们每次选择左移一步或者右移一步（但不能连续右移），每到达一个点就将答案加上这个数（包括起点）。现在一共要走kkk步，且至多能向左zzz步。求能获得的最大答案。

数据范围

1≤t≤10000,2≤k<n≤105,0≤z≤min⁡(5,k),∑n≤3×1051\leq t\leq 10000,2\leq k<n\leq 10^5,0\leq z\leq \min(5,k),\sum n\leq 3\times 10^5 1≤t≤10000,2≤k<n≤105,0≤z≤min(5,k),∑n≤3×105

题解

注意到每次向左即将答案加上连续两个数的贡献。从0−z0-z0−z枚举向左的步数，然后找出连续两个数的和最大的一对相邻的数，反复走zzz次能使答案最大。每个数据O(nk)O(nk)O(nk)。

C-Good String

题意

给定一个数字串sss，现在构造两个新串sns1s2…sn−1s_ns_1s_2\dots s_{n-1}sn​s1​s2​…sn−1​和s2s3…sns1s_2s_3\dots s_ns_1s2​s3​…sn​s1​。求最少在sss中删去多少个数可以使这两个串相等。

数据范围

1≤t≤1000,2≤∣s∣≤∑∣s∣≤2×1051\leq t\leq 1000,2\leq |s|\leq \sum |s|\leq 2\times 10^5 1≤t≤1000,2≤∣s∣≤∑∣s∣≤2×105

D-Segment Intersections

题意

有222组每组nnn条线段，第一组的每条线段都为[l1,r1][l1,r1][l1,r1]，记为aia_iai​，第二组的则为[l2,r2][l2,r2][l2,r2]记为bib_ibi​。现在每次操作可以延长任何一条线段，求使∑i=0nintersect(ai,bi)≥k\sum_{i=0}^nintersect(a_i,b_i)\ge k∑i=0n​intersect(ai​,bi​)≥k的最小操作次数。

数据范围

1≤t≤1000,1≤n≤∑n≤2×109,1≤l1,l2,r1,r2,k≤1091\leq t\leq 1000,1\leq n\leq \sum n\leq 2\times 10^9,1\leq l1,l2,r1,r2,k\leq 10^9 1≤t≤1000,1≤n≤∑n≤2×109,1≤l1,l2,r1,r2,k≤109

题解

若两条线段本身相交，则延长任意一条即可使答案+1+1+1，显然最优。所以一直操作直至所有线段对等长，然后每两次操作使答案+1+1+1，直接统计即可。

否则两个需要延长一定次数后相交，期间对相交长度无贡献，然后每次延长答案+1+1+1直至两者等长，所有等长后222次操作使答案+1+1+1，遍历1−n1-n1−n枚举使几对线段相交即可。总的复杂度为线性。

		}
	}

E-Calendar Ambiguity

题意

某纪年法中一年有mmm个月，每个月ddd天，且www天一个星期（111月111号为周一）。记(x,y)(x,y)(x,y)为第xxx个月第yyy天是星期几。求(x,y)=(y,x)(x,y)=(y,x)(x,y)=(y,x)且x<yx<yx<y的对数。

数据范围

1≤t≤1000,1≤m,d,w≤1091\leq t\leq 1000,1\leq m,d,w\leq 10^9 1≤t≤1000,1≤m,d,w≤109

题解

我们取www的剩余系考虑。则(x−1)d+y=(y−1)d+x(x-1)d+y=(y-1)d+x(x−1)d+y=(y−1)d+x，变形得：(x−y)(d−1)=0(x-y)(d-1)=0(x−y)(d−1)=0。所以(x−y)(x-y)(x−y)为wgcd(d−1,y)\frac{w}{gcd(d-1,y)}gcd(d−1,y)w​的倍数。因为1≤x,y≤min(m,d)1\leq x,y\leq min(m,d)1≤x,y≤min(m,d)，所以直接找规律然后用等差数列求和即可。常数复杂度

F-Bicolored Segments

题意

有两种颜色的线段共nnn条，现在选择这些线段的一个子集，满足异色的线段无相交。

数据范围

1≤n≤2×105,1≤li,ri≤1091\leq n\leq 2\times 10^5,1\leq l_i,r_i\leq 10^9 1≤n≤2×105,1≤li​,ri​≤109

题解

考虑线段树维护动态规划

首先，上面的做法显然是可行的，但是实在是太长了，所以采用官方提供的简单做法……

考虑建图，将异色相交的线段连边，显然原图为二分图，因此答案为n−n-n−最大匹配。

接下来考虑贪心求最大匹配。我们将一条线段[l,r][l,r][l,r]看做两个事件，从lll开始线段出现，直至rrr消失。那么根据贪心，在一条线段消失时，在对面匹配一条rrr最小的未消失线段即可。用微扰法可证明这是最优的。用set维护即可。时间复杂度O(nlog⁡n)O(n\log n)O(nlogn)。

G-Directing Edges

题意

有一张nnn个点，mmm条边的无向连通图，有kkk个指定点v1,v2,…,vkv1,v2,\dots,v_kv1,v2,…,vk​。对于每一条边，你可以选择给它指定一个方向，这不需要代价，或者使它保持双向，这需要wiw_iwi​的代价。对于每个点，若从所有指定点出发都能到达这个点，则称这个点饱和的，则你可以获得cic_ici​的代价。求对于i∈[1,n]i\in [1,n]i∈[1,n]当iii饱和时可以获得的最大的代价。

数据范围

2≤n≤3×105,n−1≤m≤min(n(n−1)2,3×105),1≤k≤n,0≤ci,wi≤1092\leq n\leq 3\times 10^5,n-1\leq m\leq min(\frac{n(n-1)}{2},3\times 10^5),1\leq k\leq n,0\leq c_i,w_i\leq 10^9 2≤n≤3×105,n−1≤m≤min(2n(n−1)​,3×105),1≤k≤n,0≤ci​,wi​≤109

题解

个人认为出的超级好的一道题。

首先，注意到对于每一个边双，其中的所有点要么同时饱和，要么同时不饱和。因为对于每一个边双，必有一种定向方法使其成为一个强连通分量。所以用TarjanTarjanTarjan缩点成一棵树。

接下来考虑进行树上dpdpdp。记fif_ifi​为当iii饱和时，iii所在的子树的最大代价。则当所有指定点都不在这棵子树中时，或所有指定点都在iii的某一个儿子中时，可以给从iii连向sonison_isoni​的边定向，同时也能获得fsonif_{son_i}fsoni​​的所有收益。前者可以通过从儿子连向爸爸，后者可以通过爸爸连向儿子满足。否则，只有不给这条边定向时，才能获得fsonif_{son_i}fsoni​​的收益，即获得max(0,fsoni−edge(i,soni))max(0,f_{son_i}-edge(i,son_i))max(0,fsoni​​−edge(i,soni​))的收益。直接dpdpdp即可。

根据上述做法，我们得到了一个n2n^2n2的做法。那么如何将其优化为O(n)O(n)O(n)呢？我们想到了”二次扫描与换根法“。我们在以111为根dpdpdp后，再次扫描整棵树，并计算出对于每个iii，当iii为根时的答案。显然，每次计算时受影响的只有iii和sonison_isoni​。所以直接深搜即可。具体的转换可以参考代码。

	if(g.edge[i]!=fa)
	{
		tmp+=dp(g.edge[i],p);
		if(num[g.edge[i]]==0||num[g.edge[i]]==k) f[p]+=f[g.edge[i]];
		else f[p]+=max(f[g.edge[i]]-g.w[i],0ll); 
	}
	num[p]=tmp;
	return tmp;
}
long long ans[MAXN];
void reroot(int p,int fa,long long cur,int e)
{
	long long tmp=f[p];
	if((num[p]!=0&&num[p]!=k)||fa==-1) cur-=max(0ll,tmp-e),tmp+=max(0ll,cur-e);
	else cur-=tmp,tmp+=cur;
	ans[p]=tmp;
	for(int i=g.hd[p];i;i=g.nxt[i])
	if(g.edge[i]!=fa) reroot(g.edge[i],p,tmp,g.w[i]);
}
int main()
{
	read(n),read(m),read(k);
	for(int i=1;i<=k;++i) read(v[i]);
	for(int i=1;i<=n;++i) read(c[i]);
	for(int i=1;i<=m;++i) read(w[i<<1]),w[i<<1|1]=w[i<<1];
	for(int i=1;i<=m;++i)
	{
		int u,v;
		read(u),read(v);
		add_edge(u,v),add_edge(v,u);
	}
	Tarjan(1,-1);
	cnt=0;
	for(int i=1;i<=n;++i)
	if(!id[i]) ++cnt,dfs(i);
	for(int i=1;i<=n;++i)
	for(int j=hd[i];j;j=nxt[j])
	if(id[i]!=id[edge[j]]) g.add_edge(id[i],id[edge[j]],w[j]);
	for(int i=1;i<=n;++i) c2[id[i]]+=c[i];
	for(int i=1;i<=k;++i) ++v2[id[v[i]]];
	dp(1,-1);
	reroot(1,-1,0,0);
	for(int i=1;i<=n;++i) write(ans[id[i]]),putchar(' ');putchar('\n');
	return 0;
}
